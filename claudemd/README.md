# claudemd — Work Journal for QC GO

This folder is the **memory of the project**. It records what was done, what
changed, and what the current state is — so you never lose track again.

## Files

| File | What it holds |
|------|---------------|
| `WORKLOG.md` | Chronological log. One dated entry per work session / change. Newest at the top. |
| `CURRENT_STATE.md` | A living snapshot of *where the project is right now* — version, what works, what's pending. Overwritten each time state changes. |
| `README.md` | This file. Explains the practice. |

## The practice (how to use this)

Every time work is done on the project:

1. **Add a `WORKLOG.md` entry** at the top with:
   - Date
   - What was done (the actions)
   - What changed (files added/edited/removed)
   - Why (the reason / goal)
   - Result (did it work? tested? pending?)
2. **Update `CURRENT_STATE.md`** so it always reflects the latest reality.
3. **Bump the version** in `CURRENT_STATE.md` when a meaningful milestone lands.

Keep entries short and honest. "Tried X, didn't work" is as valuable as a success.
